# P01 — Problem Brief

## Problem statement

ระบบแจ้งซ่อมเดิมมีเครื่องแท็บเล็ตแค่จุดเดียว พนักงานที่อยู่ไกลจึงเลือกโทรหรือบอกปากเปล่าแทน ทำให้ข้อมูลไม่ถูกบันทึกและช่างไม่รู้ว่ามีงานค้าง นอกจากนี้ทีมช่างไม่มีสิทธิ์แก้ไขข้อมูลในระบบ (เช่น เพิ่มเครื่องทดสอบ/อาการเสีย) ต้องรอทีมอื่นแก้ให้ซึ่งล่าช้ามาก ข้อมูลจึงไม่อัพเดทและไม่ตรง

## Target outcome

ระบบแจ้งซ่อมใหม่ในรูปแบบ Web Application ที่พนักงานแจ้งปัญหาได้ทันทีผ่านคอมพิวเตอร์ประจำเครื่องจักร โดยระบบจะแสดงสถานะงานซ่อมแบบเรียลไทม์บน Dashboard ในห้องช่าง และมีระบบหลังบ้านให้ทีมช่างจัดการข้อมูลระบบได้เอง 100%

## Business/User Goals

- ลดการแจ้งซ่อมตกหล่น (Zero Unrecorded Downtime): ทุกปัญหาต้องเข้าระบบ ขจัดปัญหาการแจ้งปากเปล่า

- จัดการข้อมูลเองได้: ทีมช่างสามารถเพิ่ม/ลด/แก้ไขข้อมูลเครื่องทดสอบและอาการเสียได้

- ปิดงานได้สมบูรณ์: มีระบบ Approve เพื่อยืนยันว่าเครื่องทดสอบพร้อมทำงานต่อ

- ทำงานเป็นทีม: ช่างทุกคนเห็นสถานะงานซ่อมตรงกันผ่าน Dashboard

## Primary Users

| User | Goal | Pain point |
|---|---|---|
| Operator | แจ้งซ่อมได้ทันที ไม่ต้องทิ้งเครื่องไปไกล | จุดแจ้งซ่อมเดิมอยู่ไกล ทำให้เสียเวลาเดิน |
| Technician | รับแจ้งงานรวดเร็ว ข้อมูลถูกต้อง | คนแจ้งปากเปล่าแล้วลืม, ไม่รู้ว่าช่างคนอื่นรับงานไปหรือยัง |
| Production Leader | ตรวจสอบและกดยืนยันว่าเครื่องซ่อมเสร็จจริง เพื่อรันงานต่อ | ช่างซ่อมเสร็จแล้วไม่บอก หรือบอกว่าเสร็จแต่เครื่องยังใช้ไม่ได้ |
| Technician Leader | จัดการระบบได้เอง ดูรายงานที่เชื่อถือได้ | ไม่มีสิทธิ์แก้ข้อมูลระบบเดิม รอแก้นาน |

## Success Indicators

- Adoption Rate 100%: เลิกการโทร/บอกปากเปล่า ทุกงานต้องแจ้งผ่าน Web Application

- ความเร็วในการจัดการข้อมูล: หัวหน้าช่างสามารถอัปเดตข้อมูล Master Data ได้เองภายใน 5 นาที

- Workflow สมบูรณ์: งานซ่อมถูกปิดอย่างสมบูรณ์ (พนักงานแจ้งซ่อม ➔ ช่างรับงาน ➔ ซ่อมเสร็จ ➔ Leader กด Approve)

## Assumptions

| Assumption | Impact if wrong | How to verify |
|---|---|---|
| PC หน้าเครื่องจักร กับ Server ของช่าง อยู่ IP วงเดียวกัน (หรือเชื่อม LAN ถึงกันได้) | สูง (ถ้าอยู่คนละวงเครือข่าย PC หน้าเครื่องจะเปิดเว็บแจ้งซ่อมไม่ได้เลย) | ให้ IT Test ลอง Ping ไอพีเซิร์ฟเวอร์ หรือลองเปิดหน้าเว็บจาก PC หน้าเครื่องดูว่าเข้าได้มั้ย |
| Technician Leader สามารถใช้งานระบบจัดการหลังบ้าน (Admin Panel) ได้ | ปานกลาง (ข้อมูลไม่อัปเดต) | ให้ลองใช้งานระบบต้นแบบ (Prototype) |
